## 🚀 **Quick Start**

1. **🔍 Explore** : Choisis un projet qui t'intéresse
2. **📋 Ouvre une issue** : Décris ton idée ou problème
3. **🍴 Fork** le repository
4. **🔧 Développe** ta solution
5. **🧪 Teste** tes changements
6. **📤 Crée une Pull Request**

## 📋 **Guidelines**

### **Code Quality**
- ✅ Suis les standards Python (PEP 8)
- ✅ Ajoute des tests pour les nouvelles fonctionnalités
- ✅ Documente ton code
- ✅ Utilise Black pour le formatage

### **Commits**
- 📝 Messages clairs : `feat:`, `fix:`, `docs:`, `test:`
- 🎯 Un commit = une fonctionnalité
- 📚 Référence les issues (`#123`)

### **Pull Requests**
- 📋 Description claire du changement
- ✅ Tests qui passent
- 📖 Documentation mise à jour
- 🔍 Code review friendly
